## Run 002 — 2026-07-22, CPU, d=256, epochs=10, lr=1e-3, 2 seeds

- 명령: `python main.py --domains 30 --samples 500 --seeds 2 --epochs 10 --lr 1e-3 --d 256 --d-wide 1024 --batch-size 128`

| Architecture | Final BWT | Final FWT | Experts |
|---|---:|---:|---:|
| System 2.5 (d=256) | -20.6% ± 0.8% | +2.9% ± 1.0% | 1 |
| Wide Sys 2.5 (d=1024) | -20.2% ± 0.4% | -0.3% ± 3.2% | 1 |
| System 3 | -16.4% ± 1.9% | +1.9% ± 0.6% | ~703 (폭주) |

Welch's t-test (System 3 vs Dense): BWT t=2.91, p=0.157 (2시드라 유의성 판단 불가)

**해석**:
1. ✅ 학습이 실제로 일어나면서(loss 0.4~0.9) **파괴적 망각 현상이 처음으로 실측 재현됨** — Dense BWT -20.6%는 논문의 Capacity Wall 서사(-23.4% @ d=768)와 방향·규모 일치.
2. ✅ System 3가 베이스라인 대비 BWT 우위 (방향 일치).
3. ❌ **R2P 스폰 폭주**: 라우터 projection이 task loss로 학습되며 임베딩이 이동 → 저장된 프로토타입과 유사도 하락 → 매 배치 novelty 오탐 → 전문가 700+개 스폰. 논문의 16개와 괴리.
4. ❌ Wide 모델이 Dense와 비슷한 BWT — 논문은 Wide가 덜 망각(-14.2% vs -23.4%)한다고 주장. 후속 확인 필요.

**조치**: 도메인당 스폰 예산(기본 1개) + 전문가 수 상한(기본 32) 구현 → Run 003.

## Run 003 — 2026-07-22, CPU, d=256, epochs=10, lr=1e-3, 2 seeds, 스폰 제어 적용

| Architecture | Final BWT | Final FWT | Experts |
|---|---:|---:|---:|
| System 2.5 (d=256) | -20.6% ± 0.8% | +2.9% ± 1.0% | 1 |
| Wide Sys 2.5 (d=1024) | -20.2% ± 0.4% | -0.3% ± 3.2% | 1 |
| System 3 | -21.9% ± 0.3% | +3.1% ± 0.5% | 28 |

**해석**:
1. ✅ 스폰 제어 성공: 전문가 700+ → 28개 (도메인당 ~1개).
2. ❌ 그러나 BWT가 Dense보다 악화(-21.9%). 원인 분석: **라우터 임베딩 드리프트** — projection·프로토타입이 계속 학습되므로, 과거 도메인이 최종 평가 시점에 학습 당시와 다른 전문가로 라우팅됨. 전문가 격리의 이점이 상쇄되고, 거의 매 도메인 novelty가 오탐되어 스폰 예산이 항상 소진됨(28/30).
3. **조치**: 라우터 특징(projection + 프로토타입) 동결 옵션 구현 (`freeze_features=True` 기본값). 논문 Theorem 1의 post-hoc 라우터 동결 논리를 스트림 전체로 확장한 것 — 라우팅이 x의 고정 결정적 함수가 되어 시간에 따른 라우팅 일관성이 보장됨.

## 진단 002 — 라우팅 특징 공간 분석

| 특징 공간 | 도메인 내 유사도 | 도메인 간 유사도 | 분리 가능? |
|---|---:|---:|---|
| 무작위 MLP (ReLU) | 0.996 | 0.995 | ❌ |
| 무작위 MLP (Tanh) | 0.993 | 0.992 | ❌ |
| **정규화된 원시 x (identity)** | 자기 프로토타입 0.189±0.14 (min 0.072) | 타 프로토타입 -0.001±0.014 (max 0.065) | ✅ **완전 분리** |

**조치**:
- 라우터 특징을 `identity` 모드(정규화된 원시 x)로 변경. 프로토타입 = 도메인 배치 평균 임베딩.
- τ_spawn을 측정된 분리 경계에서 캘리브레이션: **0.07** (own min 0.072 / other max 0.065 사이).
- 라우팅 온도 0.05 (유사도 스케일이 작아 sharp gating 필요 — 논문 B.3의 τ=1.0에서 벗어난 것이며, 논문 갱신 시 반영 필요 사항으로 기록).
- 첫 전문가를 lazy 스폰으로 변경 (무작위 더미 프로토타입 제거).

스모크 테스트: 4도메인 → 정확히 4전문가 (도메인당 1개, 설계 일치).

**해석**:
1. ✅ **전문가 격리가 망각을 절반 이하로 감소** (-20.6% → -9.3%) — 논문의 핵심 주장이 방향·메커니즘 면에서 실측 재현됨.
2. ✅ 전문가 13개 자율 스폰 — 논문의 16개와 유사한 스케일 (일부 도메인이 기존 전문가 공유).
3. ✅ System 3 수렴 반복수 14~20 vs Dense 28~36 — 논문의 수렴 가속 주장(§6.7)도 방향 재현.
4. ⏳ FWT는 아직 베이스라인과 차이 없음 — 남은 개선 과제.
5. 남은 격차: 논문 목표 BWT -1.8%까지는 추가 개선 필요 (EWC 강도, head 공유 완화, top-k 라우팅 정밀화 등).
